# Security Policy / 安全策略

## Supported Versions / 支持的版本

Only the latest stable release branch is supported with security patches.

## Reporting a Vulnerability / 漏洞汇报

If you identify a security vulnerability (such as a metrics spoofing exploit, alert escalation bug, dashboard credential exposure, or server-side request forgery), please do **not** open a public issue.

Please report vulnerabilities privately by emailing the maintainers or using GitHub's private vulnerability reporting feature.

如果您在系统或监控引擎中发现安全漏洞（如报警机制绕过、凭据泄露、SQL 注入或 SSRF 等），请勿通过公开 Issue 报告，请使用 GitHub 的私有漏洞报告功能或直接发送电子邮件联系维护者。
